# Cost Adjustment - Research Findings

## Window Location

Warehouse Management > Transactions > Cost Adjustment

## Core Concept

Cost Adjustment is a document-based mechanism for modifying the cost of inventory items after they have been received. Unlike Landed Cost (which distributes external costs across receipt lines), Cost Adjustment directly modifies the unit cost of specific products based on identified discrepancies, price corrections, or retroactive cost changes.

## Table Structure

### M_CostAdjustment (Header)

The document header. Contains the document lifecycle fields and organizational context. Acts as a container for adjustment lines.

Key columns:
- `DocumentNo` -- auto-generated sequence
- `DateAcct` -- accounting date for the cost adjustment postings
- `DocStatus` -- standard document lifecycle (DR -> CO -> VO)
- `C_DocType_ID` -- cost adjustment document type
- `Processed` -- flag set on completion
- `Description` -- optional free text
- `ReferenceNo` -- external reference number

### M_CostAdjustmentLine (Detail Lines)

Each line represents a cost modification for a specific product, optionally linked to a material transaction.

Key columns:
- `M_Product_ID` -- FK to M_Product; the product whose cost is being adjusted
- `M_Transaction_ID` -- FK to M_Transaction; the specific inventory transaction being adjusted (optional)
- `AdjustmentAmount` -- the amount to add/subtract from the product cost (positive = increase, negative = decrease)
- `IsSource` -- boolean flag indicating this is a manually entered (source) adjustment line
- `IsRelated` -- boolean flag indicating this is a system-generated cascading adjustment (e.g., BOM propagation)
- `Line` -- line number for ordering
- `C_Currency_ID` -- currency of the adjustment amount
- `Description` -- optional description for the line

## Relationship to Inventory

```
M_CostAdjustment (header)
  |
  +-- M_CostAdjustmentLine (adjustment lines)
        |
        +-- M_Product_ID -> M_Product (product being adjusted)
        +-- M_Transaction_ID -> M_Transaction (specific inventory transaction)
        |
        +-- On Process: updates M_CostingRule / M_Product_Costing
```

## Source vs Related Lines

Cost adjustments distinguish between two types of lines:
- **Source lines** (`IsSource = true`): Manually entered by the user. These are the primary cost corrections.
- **Related lines** (`IsRelated = true`): Auto-generated by the system when a source adjustment triggers cascading effects (e.g., adjusting a raw material cost propagates to BOM-manufactured products).

A line can be both source and related in edge cases, but typically it is one or the other.

## Document Lifecycle

1. **Draft (DR):** User creates header, adds adjustment lines with products and amounts.
2. **Complete (CO):** System validates lines, applies adjustment amounts to product costing records, creates accounting entries for cost variance.
3. **Void (VO):** Reverses all cost adjustments and undoes product cost changes.

## Key Differences from Landed Cost

- **Direct adjustment:** Cost Adjustment modifies product cost directly, while Landed Cost distributes external costs proportionally across receipt lines.
- **No distribution method:** Each line specifies an exact adjustment amount. There is no proportional distribution calculation.
- **Transaction-level granularity:** Lines can reference specific M_Transaction records, enabling precise cost corrections for individual receipts.
- **Cascading support:** The Source/Related mechanism supports automatic propagation of cost changes through BOM structures.
- **Simpler structure:** Two entities (header + lines) vs Landed Cost's four entities.

## Schema Design Decisions

1. **Two entities:** Header (M_CostAdjustment) and lines (M_CostAdjustmentLine). Simpler than Landed Cost.
2. **Organization editable on header:** Allows selection of the organizational unit for the adjustment.
3. **IsSource and IsRelated as readOnly:** These flags are managed by the system, not by the user. IsSource defaults to true for user-created lines; IsRelated is set by cascading logic.
4. **Transaction reference optional:** A line can adjust product cost without referencing a specific transaction (general adjustment) or target a specific transaction (precise correction).
5. **Currency on lines:** Each line can specify the currency of the adjustment amount, supporting multi-currency environments.
