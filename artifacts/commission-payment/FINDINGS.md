# Commission Payment - Research Findings

## Tables

- **Header:** `C_CommissionRun` -- a single commission calculation run
- **Lines:** `C_CommissionAmt` -- calculated commission amounts per invoice line

## Window Identity

- **Window ID:** 176 (Sales Management > Transactions > Commission Payment)
- **Parent Definition:** `C_Commission` defines the commission contract (sales rep, percentage, frequency). `C_CommissionRun` is a specific execution of that contract over a date range.
- **Document types:** Commission Run (no dedicated doc type -- uses generic processing)

## Entity Relationships

```
C_Commission (commission contract definition)
    |
    v  [Calculate Commission process]
C_CommissionRun (one run per period)
    |
    v
C_CommissionAmt (one line per qualifying invoice line)
    |
    v
C_InvoiceLine (reference to the invoiced sale that generated the commission)
```

## Key Concepts

### Commission Contract (C_Commission)

The `C_Commission` table defines:
- Which sales representative earns commission
- Commission percentage or fixed amount
- Frequency (monthly, quarterly, etc.)
- Which products/product categories qualify
- Minimum and maximum thresholds

This table is NOT part of the Commission Payment window -- it is a separate master data window. The Commission Payment window only references it via `C_Commission_ID` on the run header.

### Commission Run (C_CommissionRun)

A commission run represents one calculation cycle:
- `StartDate` / `EndDate` define the period being calculated
- `C_Commission_ID` links to the commission contract
- `GrandTotal` is the sum of all `C_CommissionAmt.CommissionAmt` lines
- `DocumentNo` is auto-generated
- `DocStatus` follows standard document lifecycle (DR -> CO -> VO)

### Commission Amount (C_CommissionAmt)

Each line represents a calculated commission for one invoice line:
- `C_InvoiceLine_ID` -- the invoice line that triggered the commission
- `CommissionAmt` -- the calculated commission amount
- `ActualQty` -- the quantity from the invoice line
- `ActualAmt` -- the net amount from the invoice line
- `ConvertedAmt` -- the amount converted to the commission contract's currency

Lines are **generated by the calculation process**, not entered manually. They are always read-only in the UI.

## Key Processes

1. **Calculate Commission**: Runs against a commission contract for a date range. Scans qualifying invoices, calculates amounts per the contract rules, and creates `C_CommissionAmt` lines.
2. **Complete (DocAction)**: Marks the run as processed. Prevents further modification.
3. **Void**: Reverses a completed commission run.

## Schema Decisions

- **Commission Amount lines are all readOnly** -- they are output of the calculation process, not user-editable
- **No manual line creation** -- lines are generated programmatically by the Calculate Commission process
- **commission** (C_Commission_ID) is editable on header -- user selects which commission contract to run
- **startDate** is editable and required -- defines the calculation period start
- **endDate** is editable but optional -- if null, calculates up to current date
- **grandTotal** is readOnly -- sum of all commission amount lines
- **documentNo** is readOnly -- auto-generated sequence
- **docStatus** is readOnly -- managed by document action process

## System Fields Pattern

Follows the standard system columns from `core-maps/system-columns.json`:
- `AD_Client_ID`, `AD_Org_ID` from context
- `Created`, `Updated` timestamps
- `Processed` derived from `DocStatus`

## Commission Amount Line Inheritance from Header

Lines inherit from header (via `fromParent` derivation):
- `C_CommissionRun_ID` (parent FK)
