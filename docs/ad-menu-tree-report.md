## SQL Queries Used

### 1. Full Menu Tree (recursive CTE)

```sql
WITH RECURSIVE menu_tree AS (
  -- Root nodes
  SELECT
    tn.node_id,
    tn.parent_id,
    tn.seqno,
    m.name,
    m.issummary,
    m.action,
    m.ad_window_id,
    m.ad_process_id,
    m.ad_form_id,
    m.em_obuiapp_process_id,
    0 AS depth,
    m.name::text AS path
  FROM ad_treenode tn
  JOIN ad_menu m ON m.ad_menu_id = tn.node_id
  WHERE tn.ad_tree_id = '10'
    AND tn.parent_id = '0'
    AND m.isactive = 'Y'

  UNION ALL

  -- Children
  SELECT
    tn.node_id,
    tn.parent_id,
    tn.seqno,
    m.name,
    m.issummary,
    m.action,
    m.ad_window_id,
    m.ad_process_id,
    m.ad_form_id,
    m.em_obuiapp_process_id,
    mt.depth + 1,
    mt.path || ' > ' || m.name
  FROM ad_treenode tn
  JOIN ad_menu m ON m.ad_menu_id = tn.node_id
  JOIN menu_tree mt ON tn.parent_id = mt.node_id
  WHERE tn.ad_tree_id = '10'
    AND m.isactive = 'Y'
)
SELECT
  repeat('  ', depth) ||
  CASE
    WHEN issummary = 'Y' THEN '[FOLDER] '
    WHEN action = 'W' THEN '[WINDOW] '
    WHEN action = 'P' OR action = 'R' THEN '[PROCESS/REPORT] '
    WHEN action = 'X' THEN '[FORM] '
    WHEN action = 'S' THEN '[WIDGET] '
    ELSE '[' || COALESCE(action, '?') || '] '
  END || name AS menu_tree
FROM menu_tree
ORDER BY path, seqno;
```

### 2. Item Type Distribution

```sql
SELECT
  CASE
    WHEN m.issummary = 'Y' THEN 'Folder'
    WHEN m.action = 'W' THEN 'Window'
    WHEN m.action = 'P' THEN 'Process'
    WHEN m.action = 'R' THEN 'Report'
    WHEN m.action = 'X' THEN 'Form'
    WHEN m.action = 'S' THEN 'Widget'
    ELSE COALESCE(m.action, 'Unknown')
  END AS type,
  count(*) AS total
FROM ad_treenode tn
JOIN ad_menu m ON m.ad_menu_id = tn.node_id
WHERE tn.ad_tree_id = '10'
  AND m.isactive = 'Y'
GROUP BY 1
ORDER BY 2 DESC;
```

### 3. Depth Distribution

```sql
WITH RECURSIVE menu_tree AS (
  SELECT tn.node_id, tn.parent_id, 0 AS depth
  FROM ad_treenode tn
  JOIN ad_menu m ON m.ad_menu_id = tn.node_id
  WHERE tn.ad_tree_id = '10'
    AND tn.parent_id = '0'
    AND m.isactive = 'Y'
  UNION ALL
  SELECT tn.node_id, tn.parent_id, mt.depth + 1
  FROM ad_treenode tn
  JOIN ad_menu m ON m.ad_menu_id = tn.node_id
  JOIN menu_tree mt ON tn.parent_id = mt.node_id
  WHERE tn.ad_tree_id = '10'
    AND m.isactive = 'Y'
)
SELECT depth AS level, count(*) AS items
FROM menu_tree
GROUP BY depth
ORDER BY depth;
```

### 4. Top-Level Folder Sizes

```sql
WITH RECURSIVE menu_tree AS (
  SELECT
    tn.node_id, m.name, m.issummary,
    tn.node_id AS root_id, m.name AS root_name
  FROM ad_treenode tn
  JOIN ad_menu m ON m.ad_menu_id = tn.node_id
  WHERE tn.ad_tree_id = '10'
    AND tn.parent_id = '0'
    AND m.isactive = 'Y'
    AND m.issummary = 'Y'
  UNION ALL
  SELECT
    tn.node_id, m.name, m.issummary,
    mt.root_id, mt.root_name
  FROM ad_treenode tn
  JOIN ad_menu m ON m.ad_menu_id = tn.node_id
  JOIN menu_tree mt ON tn.parent_id = mt.node_id
  WHERE tn.ad_tree_id = '10'
    AND m.isactive = 'Y'
)
SELECT root_name AS top_folder, count(*) - 1 AS children
FROM menu_tree
GROUP BY root_id, root_name
ORDER BY children DESC;
```

## Table Relationships

```
ad_tree (id=10, type='MM')
    │
    └─── ad_treenode (parent_id / node_id hierarchy)
              │
              └─── ad_menu (the actual menu item)
                      │
                      ├── issummary = 'Y' → Folder
                      ├── action = 'W'    → Window (→ ad_window)
                      ├── action = 'P'    → Process (→ ad_process)
                      ├── action = 'R'    → Report (→ ad_process)
                      ├── action = 'X'    → Form (→ ad_form)
                      └── action = 'OBUIAPP_Process' → (→ obuiapp_process)
```

- `ad_tree` with `treetype = 'MM'` defines the menu tree (id `10`)
- `ad_treenode` stores the parent-child relationships via `parent_id` and `node_id`
- Root nodes have `parent_id = '0'`
- `ad_menu.issummary = 'Y'` marks folders (grouping nodes with no action)
- `ad_menu.action` determines the item type (W=Window, P=Process, R=Report, X=Form, etc.)
