# App Shell Design — Production Frontend Scaffold

> **Date:** 2026-03-05
> **Status:** Approved
> **Goal:** Create the production app shell where Schema Forge generated UIs mount. Login against Etendo, sidebar navigation, React Router per window.

---

├── index.html
├── package.json
├── vite.config.js

│   ├── auth/

│   ├── windows/

│       └── utils.js             (shadcn cn() utility)
```

## Auth Flow

1. User lands on `/login`
2. `LoginPage` renders a form (username + password)
3. On submit: `POST /etendo/sws/login` with `{ user, password }`
4. Response: `{ token: "jwt..." }`
5. Token stored in `AuthContext` (React state) + `localStorage`
6. All API requests via `api.js` include `Authorization: Bearer <token>`
7. On 401 response → clear token, redirect to `/login`
8. On logout → clear token + localStorage, redirect to `/login`

│  TopBar (logo, user info, logout)        │
├────────────┬─────────────────────────────┤
│            │                             │

│            │   (React Router <Outlet />)  │

```

- **Sidebar**: Shadcn Sidebar component, menu items read from contract.json
- **Routing**: `/{window-name}` → loads the corresponding generated component
- **Default route**: redirects to first window in contract

## Generated Component Contract

Components generated by `/sf:generate-ui` receive standard props from the shell:

```jsx
<GeneratedWindow
  token={authToken}           // JWT for API calls
  apiBaseUrl="/etendo/api"    // backend base URL
  window={windowConfig}       // window metadata from contract
/>
```

Generated components must:
- Accept `{ token, apiBaseUrl, window }` as props
- Use `apiBaseUrl` + `token` for all fetch calls (no hardcoded URLs)
- Export default a standard React component
- Use inline styles + base React (no dependency on shell's Shadcn/Tailwind)

## Routing Structure

```
/login              → LoginPage (public)
/                   → redirect to first window
/:windowName        → WindowLoader → generated component
```

Auth guard: all routes except `/login` require a valid token in AuthContext.

## Menu Source

Static from `contract.json`:
- Read `frontendContract.entities` keys
- Each entity becomes a menu item
- Label from entity display name, icon optional
- Route: `/{entity-name}`

## Window Loading

`WindowLoader.jsx`:
1. Reads `windowName` from route params
2. Dynamically imports from `artifacts/{window}/generated/web/{window}/{entityName}.jsx`
3. Passes `{ token, apiBaseUrl, window }` props
4. Shows loading state while importing
5. Shows error state if component not found

## What's NOT Included (YAGNI)

- Token refresh (user re-logs on expiry)
- i18n
- Themes / dark mode
- Breadcrumbs
- Toast notifications
- State management library (no Redux/Zustand)
- Offline support
- Role-based menu filtering (all windows visible)
