Por Que QA es Critico en SaaS

En on-premise, un bug afecta a un cliente. En SaaS, a todos al mismo tiempo.

Sin Tests
Deploy manual con miedo.
Bugs descubiertos por el cliente.
Rollback tardio y doloroso.
Con Cobertura
Deploy automatico con confianza.
Bugs detectados antes de produccion.
Delivery continuo habilitado.

Sin cobertura de tests, no se deploya.

Deployment — Modulo Versionado

moduleVersion
Se incrementa en cada regeneracion.
Le importa al pipeline de deploy.
apiVersion
Se incrementa cuando cambia el DTO.
Le importa al frontend.
behavioralVersion
Se incrementa cuando cambian reglas.
Le importa a los tests.

Regenerar UI sin romper APIs. Agregar una regla sin forzar update del frontend.

Deployment — Que se Entrega

com.etendo.go/                    # Un unico modulo (nombre WIP)
├── event/          # Event handlers
├── process/        # Procesos
├── dto/v{n}/       # DTOs versionados
├── api/v{n}/       # Endpoints REST versionados
├── mapper/v{n}/    # OBDal ↔ DTO
└── web/            # React SPA (todas las ventanas)

Un solo modulo contiene todas las ventanas generadas. Se instala con Gradle.

Cada regeneracion produce un artefacto desplegable. Sin pasos manuales. Sin builds rotos.

Infraestructura — Como lo Logramos

Observabilidad

  • Monitoreo de salud (tiempos de respuesta, errores)
  • Logging estructurado + agregacion
  • Alertas con cadenas de escalamiento

Auto-Recovery

  • Orquestacion de containers + health checks
  • Reinicio automatico ante fallos
  • Degradacion elegante bajo carga

Deploy Seguro

  • Blue-green deploys (zero downtime)
  • Rollback automatico ante health checks fallidos
  • Migraciones de DB versionadas y reversibles

Disaster Recovery

  • Backups automaticos (diarios + incrementales)
  • Recuperacion point-in-time
  • Runbooks documentados + simulacros de DR

Preguntas Frecuentes

Todavia necesito saber Java?
Si. Revisas codigo generado y manejas casos borde.
Que pasa si el codigo generado esta mal?
El pipeline lo atrapa: DEV → REVIEW → QA. Los contract tests detectan problemas al instante.
Y las ventanas existentes en produccion?
Se extraen y regeneran incrementalmente. No hace falta una migracion big-bang.
